本文件为 Codex (Codex.ai/code) 在此代码仓库中工作时提供指导。

## 项目概述

这是一个针对中国两种彩票游戏的号码预测与分析系统：
- **双色球 (SSQ)**: 7个红球 (01-33) + 1个蓝球 (01-16)
- **大乐透 (DLT)**: 5个前区号码 (01-35) + 2个后区号码 (01-12)

系统通过分析历史开奖数据，应用多策略融合方法来推荐下期号码。

## 技术栈

- **语言**: Python 3（通过 `D:\anaconda3\python.exe` 运行）
- **数据处理**: pandas、xlsx 等库
- **输入数据**: Excel文件格式（.xlsx）

## 核心选号策略

### 双色球 (SSQ) 策略

**红球7码结构**:
- 2-3个历史重组骨架号（来自最近50期的高频对子/三数组合）
- 2个近邻斜连号（最近1-3期的±1承接）
- 1个多期斜连号（等差递进模式）
- 1个即时跳点号（模型排名8-15位，防止过度优化）
- 可选：1个结构平衡号/区带回摆号

**硬性约束**:
- 低区（01-11）：优先2个
- 中区（12-22）：优先2-3个
- 高区（23-33）：优先1-3个
- 奇偶比例：3:4 或 4:3
- 不允许4个以上号码来自同一逻辑来源
- 不允许原样照搬历史某期6红组合

**蓝球选号**（必须同时参考4个维度）:
1. 最近10期蓝球活跃度
2. 近邻斜连（±1）
3. 波动回补
4. 历史区间冷热平衡

### 大乐透 (DLT) 策略

**前区5码结构**（按优先级分层）:

| 层级 | 类型 | 数量 | 说明 |
|------|------|------|------|
| 第一层 | 最近1期承接号 | 1-2个 | 首选：重号、±1近邻、密集段外扩 |
| 第二层 | 历史重组骨架号 | 1-2个 | 来自最近50期高频组合 |
| 第三层 | 辅助斜连号 | 0-1个 | 近期±1或多期等差斜连 |
| 第四层 | 逆向跳号 | 1个 | 必须有，模型前20名以外的补位号 |
| 第五层 | 结构平衡号 | 0-2个 | 修正奇偶/连号/数段分布 |

**硬性约束**:
- 低区（01-12）：优先1个，允许没有（软约束）
- 中区（13-24）：优先1个
- 高区（25-35）：允许没有（断区时）
- 奇偶比例：2:3 或 3:2
- 不允许4个以上号码全部来自同一条斜连逻辑
- 不允许整张票原样照搬历史某期

**后区2-3码**:
- 1个活跃延续号（最近3-5期反复出现）
- 1个斜连号（±1承接）
- 可选：1个回补号（短期沉寂后的回补窗口）

## 重要规则补充

### 断区识别（大乐透重点）

当满足以下任意2条时，视为"强断区信号"：
- 最近6期内出现2次及以上完整断区
- 上一期三区分布极端偏斜（如0:1:4、4:1:0、3:2:0）
- 排序定位出现异常上跳或下沉

**放行规则**：允许单票采用"某一区为0个"的结构

### 极端结构后的反抽（大乐透）

- 若上一期为极端偏态，下一期必须准备1注反抽票
- 反抽票优先结构：2:1:2、2:2:1 或 1:2:2
- 多票时偏态延续票与反抽票必须并存

### 准断区/断区（双色球）

- 准断区：某区只开1个号码
- 完整断区：某区开0个号码
- 最近6期断区≥2次或准断区≥4次时，放行准断区结构

## 核心原则

1. **多策略融合**：任何单一策略都不应主导最终选号
2. **斜连降级**：仅作为辅助信号，不作为主定号逻辑
3. **历史重组**：用于验证结构合理性，而非机械复刻旧号码
4. **强制多样性**：必须包含"跳点号"或"逆向跳号"以防止过拟合
5. **赛后复盘**：每次开奖后分析未中原因，持续优化策略

## 实现要点

1. 为双色球和大乐透维护**独立的分析管道**
2. 各策略层**独立实现**，最后才做约束校验
3. 生成多个候选集并排序，而非只输出单一结果
4. 硬性约束作为**最终验证步骤**，不是前置过滤
5. 包含**赛后复盘功能**追踪预测准确度

## 复盘检查项

每次开奖后必须回答：
1. 实际号码中有几个原本在模型前列？
2. 没中的号是候选池漏掉了，还是收口删错了？
3. 是否出现断区/准断区，而被均衡思维过滤掉了？
4. 多票之间是否只是换壳，没有真正分散主线？
5. 蓝球/后区问题是模型错误还是策略太死？
