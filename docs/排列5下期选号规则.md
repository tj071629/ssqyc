# 排列5下期选号规则（逐期自适应版）

## 游戏与数据口径

- **玩法**：排列5为 5 位数字直选，每位 0–9，共 10^5 种组合；顺序完全固定，与排列3一致为**定位玩法**。
- **历史数据**：`docs/p5.md`，表格列为「开奖期号｜开奖日期｜开奖号码」，号码为连续 5 位数字字符串（可含首位 0）。
- **默认建模口径**：凡基于 `src/pl5_rolling_backtest.py` 的打分、候选排序与滚动回测，一律以 **逐期学习后的权重**（每期开奖后按脚本内 `update_state` 逻辑更新 `StrategyState`）为准；仅在需要「固定规则对照」时，再并列运行 `--fixed` 固定权重模式。

## 自我修改学习（逐期权重更新）在做什么

模型把每一注 5 位号码映射为一组 **0–1 区间的特征**，再用加权求和得到总分；总分仅用于在 10 万种直选中**排序**，不表示概率或「必中」。

每期流程：

1. 仅用**当期之前**的全部历史，构造统计量（冷热、转移、遗漏、结构分布等）。
2. 对 10 万种号码批量打分并排序，取前 N 注作为当期「候选池」。
3. 开奖后，读取**真实开奖号码**对应的特征向量，并与当期排名前 `update_top_n`（默认 20）注的特征做对比，按与排列3相同的规则更新各特征权重：在「锚定初始权重」的基础上，向「开奖特征」方向小幅修正，并做上下限裁剪。

因此：**学习的是各信号的相对重要性（权重），不是改写历史开奖**；规则文件描述的是**信号含义与使用顺序**，具体权重数值以最近一次脚本输出（回测报告中的「当前权重」表）为准。

### 「不断完善选号规则」在本项目里的分工

- **逐期自适应（已实现）**：在**固定的一组特征信号**上，每期开奖后微调权重，使排序随近期数据缓慢漂移；这是可复现、可回测的「自我修改」。
- **完善规则本身（特征/逻辑）**：属于**改代码或改本文件信号定义**，需要设计、留出验证集或分段滚动对比，**不能**靠每期自动改文案或盲目加条件，否则容易全历史过拟合。若要加新信号，应在 `pl5_rolling_backtest.py` 中增加特征并在回测里验证，再同步更新本表。

## 特征信号（与代码字段名一致）

| 字段名 | 含义 |
| --- | --- |
| `pos_hot10` / `pos_hot30` / `pos_hot100` | 各位置在最近 10、30、100 期内的数字频率，按位取平均并归一。 |
| `all_hot30` | 最近 30 期全体位置上数字出现频率，对 5 个位置上的数字取平均并归一（分母含每期 5 个数字）。 |
| `repeat_last` | 与上一期同位是否相同（重号），按位占比。 |
| `neighbor_last` / `neighbor_prev` | 与上一期、上上期同位是否为 ±1 邻号，按位占比。 |
| `transition` | 近约 250 期内，同位「上一期数字 → 候选数字」的转移计数相对强度，五位平均。 |
| `omission_mid` | 各位置候选数字当前遗漏期数，套中等窗口高斯型得分后取平均。 |
| `omission_deep` | 深遗漏线性压缩得分，五位平均。 |
| `sum_hot100` | 5 位之和落在近 100 期和值分布中的相对频率。 |
| `span_hot100` | 跨度（最大减最小）在近 100 期分布中的相对频率。 |
| `pattern_hot100` | 不同数字个数形态 `u1`–`u5`（五位全同记为 u1，五位互异记为 u5）在近 100 期分布中的相对频率。 |
| `even_odd_fit` | 偶数位个数（0–5）在近 100 期分布中的相对频率。 |

## 如何生成预测与复盘材料

在项目根目录执行（Python 见本机环境，仓库说明中常用 `D:\anaconda3\python.exe`）：

```text
python src/pl5_rolling_backtest.py --data docs/p5.md --report docs/排列5逐期自适应回测报告.md
```

固定权重对照：

```text
python src/pl5_rolling_backtest.py --data docs/p5.md --report docs/排列5固定权重滚动回测报告.md --fixed
```

回测报告含：数据范围、命中率、位置命中分布、实际开奖在排序中的名次统计、**当前权重**、**下一期前 20 候选**等。其中「下一期候选」仅反映**截至历史最后一期**的模型状态；新开奖后需重新跑脚本以滚动更新。

回测报告另含 **「下期三期分段各一注」**：在模型排名前 100 注（可用 `--stratified-pool N` 修改池大小）中按名次三等分，每段取该段**最靠前**的一注，对应全排序约第 **1、34、67** 名（当池为 100 时）。该取法用于分散注单，不表示提高中奖概率。

每期更新 `docs/p5.md` 后，若只需在终端看摘要与上述三注（仍写 Markdown 报告、不刷屏）：

```text
python src/pl5_rolling_backtest.py --data docs/p5.md --report docs/排列5逐期自适应回测报告.md --summary-only
```

另取模型 Top N 连续预测列表：

```text
python src/pl5_rolling_backtest.py --data docs/p5.md --report docs/排列5逐期自适应回测报告.md --summary-only --predict-top 3
```

若预测列表中**排除历史上已开出过的直选号**（在 10 万种中顺延）：

```text
python src/pl5_rolling_backtest.py --data docs/p5.md --report docs/排列5逐期自适应回测报告.md --summary-only --predict-top 3 --never-drawn-only
```

若需「相对留出段」比较不同学习超参（**不**自动发明新特征，只在固定信号上搜 `learning_rate` / `decay`）：

```text
python src/pl5_rule_tuner.py --data docs/p5.md --holdout 300 --quick
```

仅用最近若干期做快速实验（缩短单次运行时间，结论更噪）：

```text
python src/pl5_rule_tuner.py --data docs/p5.md --tail 4000 --holdout 250 --quick --out docs/排列5留出超参搜索.md
```

输出 Markdown 中含：留出段 **直选一等奖（每期只跟模型第 1 注，排名=1）次数**、**最佳排名**，以及各超参方案对比表。

## 选号策略层面的硬性提醒

1. **直选空间极大**：随机单注期望命中率为 1/100000；任何回测若仅略高于随机，应视为噪声范围内波动，不得据此加杠杆投注。
2. **多信号融合**：单一信号（例如只看遗漏或只看热号）不应主导；权重学习的目的正是抑制某一信号长期虚高。
3. **防过拟合**：避免根据某一短窗段的命中表现，手工把本文件中的文字规则改得越来越细；若需调参，应固定验证窗（例如留出最近若干期）或做分段滚动回测，再比较指标。
4. **复盘顺序**：先看「开奖号在模型中的排名」与「候选池宽度」，再区分是排序模型问题还是资金管理问题；排列5无组选等价口径，直选即最终评价标准。

## 与本文件的关系（避免「规则正文」与程序脱节）

- **本文件**：约定玩法、信号定义、学习机制与使用方式；不随每期开奖自动变更。
- **程序与回测报告**：输出每期滚动后的**数值权重**与候选排序；若需「不断提升」建模质量，应通过**改代码特征/验证方案/超参**并在报告里留痕，而不是在本文中堆叠未经回测证实的断言性语句。
