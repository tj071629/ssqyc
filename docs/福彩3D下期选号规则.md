# 福彩3D下期选号规则（回测修正版）

## 游戏与数据口径

- **玩法**：三位数字 **000–999**，顺序固定；常见投注为 **直选**（2 元/注）、**组三**（约 1/3 组合空间）、**组六**（约 1/6 组合空间）。
- **历史数据**：`docs/3d.md`，表格列为「开奖期号｜开奖日期｜开奖号码」，3 位数字连写（如 `267`）。
- **组合空间**：直选 **1000** 种，理论直选中奖率 **1/1000**。
- **默认建模口径**：基于 `src/fc3d_rolling_backtest.py`（引擎与 `src/p3_rolling_backtest.py` 相同）的多信号融合打分；**默认使用固定权重**（`INITIAL_WEIGHTS`），逐期自适应仅作对照。

### 奖级匹配（福彩3D直选）

| 奖级 | 条件 |
| --- | --- |
| 单选（直选） | 三位数字及顺序全同 |
| 组选三 | 三位数字多重集合相同，且形态为组三（两同） |
| 组选六 | 三位数字多重集合相同，且形态为组六（三不同） |

组选奖金与投注方式相关；本文「组选宽口径」仅用于回测观察数字是否进池，**不等于**可兑现的组选奖金。

## 滚动回测结论（8538 期，2002101–2026130）

以下结论来自 `src/fc3d_rolling_backtest.py`、`src/fc3d_strategy_compare.py`，**须在每期更新 `docs/3d.md` 后重跑脚本复核**。

### 1. 直选：规则无法稳定战胜随机

| 策略 | 1 注命中率 | 5 注 | 10 注 | 20 注 | 实际号均排名 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 随机 | 0.094% | 0.515% | 1.078% | **2.061%** | — |
| 近 30 期按位热号 | **0.176%** | 0.176% | 0.176% | 0.176% | — |
| 融合固定权重 | 0.141% | 0.504% | 1.101% | 1.897% | 498.7 / 1000 |
| 融合逐期自适应 | 0.105% | 0.621% | 1.054% | 2.038% | 501.3 / 1000 |

**结论**：

- 单注直选：近 30 期按位热号略优于融合 Top1，但样本内优势很小，不可当作稳定超额。
- 5 注以上：融合模型与随机在理论值附近波动；20 注时随机 **2.061%** 仍不低于融合固定 **1.897%**。
- 实际开奖号落入模型前 20 约 **1.9%**（理论随机亦为 2%），说明融合打分主要是 **排序**，不是预测。

### 2. 固定权重 vs 逐期自适应

| 指标 | 固定权重 | 逐期自适应 |
| --- | ---: | ---: |
| 1 注直选 | **0.141%** | 0.105% |
| 20 注直选 | 1.897% | **2.038%** |
| 20 注组选宽口径 | **10.810%** | 8.925% |
| 实际号前 100 率 | **9.979%** | 9.569% |

**推荐**：日常以 **固定权重** 为准；若多注（≥10）可并列参考自适应 Top20，但 **不要** 指望自适应在组选宽口径上更优。

### 3. 按位热号：适合单注，不适合扩复式

- 近 30 期每位取频率最高数字拼成 1 注（如当前 **260**）。
- 全量 8538 期：单注 **0.176%**，高于融合 Top1 的 0.141%。
- 但热号只有 1 注，**无法**随票数扩展；5/10/20 注命中率与 1 注相同。
- 按位 Top1 单独命中率约 **10%**（与随机理论一致），三位全中仍约 **1/1000**。

### 4. 结构分布（近 100 期参考，2026130 同期）

- 形态：组六 **75** / 组三 **25** / 豹子 **0**
- 跨度 Top3：7（17 次）、4（15 次）、6（15 次）
- 偶数个数：2 个偶数最常见（44 次）
- 和值 Top3：15、17、10

选号时优先 **组六形态**，和值 10–17、跨度 4–7 作软约束；豹子不作为主追方向。

### 5. 近窗口波动（勿过拟合）

| 窗口 | 热号 1 注 | 固定 20 注 | 随机 20 注 |
| --- | ---: | ---: | ---: |
| 近 500 期 | 0.600% | 2.000% | 1.600% |
| 近 200 期 | 0.500% | 3.000% | 1.500% |
| 近 50 期 | 0.000% | 6.000% | 0.000% |

短窗口命中率方差极大；**不得以近 50 期 6% 反推长期有效**。

## 特征信号（与 `p3_rolling_backtest.py` / `fc3d_rolling_backtest.py` 一致）

对 000–999 全候选打分排序，按位与整票结构联合建模。

| 字段名 | 含义 |
| --- | --- |
| `pos_hot10` / `pos_hot30` / `pos_hot100` | 各位置在最近 10、30、100 期数字频率。 |
| `all_hot30` | 三位数字在最近 30 期整体出现频率。 |
| `repeat_last` | 与上期同位重号。 |
| `neighbor_last` / `neighbor_prev` | 与上期、上上期同位 ±1 邻号。 |
| `transition` | 近约 250 期同位转移频率（上期数字 → 候选数字）。 |
| `omission_mid` / `omission_deep` | 同位遗漏的中窗 / 深窗得分。 |
| `sum_hot100` / `span_hot100` / `pattern_hot100` / `even_odd_fit` | 和值、跨度、组三/组六/豹子、偶数个数在近 100 期的结构偏好。 |

**初始权重**见脚本内 `INITIAL_WEIGHTS`；3D 全量回测支持 **固定此组权重**，不必每期 `update_state`。

## 推荐选号流程（实操）

### A. 单注直选（2 元）

1. 取 `docs/3d.md` 表末向上最近 **30 期**。
2. **百/十/个位**各取出现次数最多的数字；并列时优先 **上期同位重号**，其次 **±1 邻号**。
3. 用 `fc3d_rolling_backtest.py --fixed` 的融合固定权重 **Top1** 再出 1 注。
4. 两注若相同只保留 1 注；若不同 **各买一注**，避免两条逻辑被同一号码绑死。

**当前示例（基于 2026130 后）**：

| 来源 | 号码 |
| --- | --- |
| 近 30 期按位热号 | **260** |
| 融合固定权重 Top1 | **154** |

### B. 小复式直选（5–20 注）

1. **禁止**只取模型排名第 1–N 的连续号码（回测 20 注 ≤ 随机）。
2. 在融合 Top **100** 内 **分段取号**：如第 1、12、25、38、51、64、77、90 名，并强制加入 **排名第 8–15 位** 的 1 注跳点号。
3. 至少 1 注与 Top1 **形态不同**（Top1 偏组六则补 1 注组三结构候选）。
4. 5 注以内可 3 注模型分散 + 2 注随机；10 注以上随机占比不低于 **30%**。

### C. 组选（组三 / 组六）

1. 先用 B 的流程生成 5–10 个直选候选。
2. 按 `sorted(三位)` 去重，保留 **组六优先**（覆盖 6 个直选排列）或 **组三**（覆盖 3 个排列）。
3. 宽口径回测：固定权重 5 候选组选约 **3.09%**，接近理论覆盖 **2.51%**；20 候选约 **10.8%**，与理论 **9.3%** 相当。
4. **不要**只买 1 注组选并指望长期超理论；组选优势来自 **覆盖数字集合**，不是单点预测。

### D. 结构硬约束（最终校验）

- 和值建议 **8–20**（近 100 期主区）；极端和值仅作跳号保留 1 注。
- 跨度建议 **3–8**；全大/全小（如 789、012）最多 1 注。
- 奇偶：优先 **1:2 或 2:1**；全奇/全偶最多 1 注。
- 形态：组六为主；连续 3 期组三后可补 1 注组三，但不连续追组三。
- **禁止**原样照搬历史某期直选号；**禁止** 4 注以上全部来自同一信号（如全是邻号票）。

## 多票搭配（≥2 注时）

| 票种 | 结构 | 说明 |
| --- | --- | --- |
| 热号直选 | 近 30 期按位热号 | 单注基准 |
| 融合主票 | 固定权重 Top1–Top3 分散 1 注 | 模型主线 |
| 跳点票 | 排名第 8–15 | 防过拟合，**必须有** |
| 结构平衡 | 与主票形态/和值/跨度错开 | 修正偏态 |
| 随机票 | 1–2 注均匀随机 | 20 注组合时不可省略 |

### E. 单选复式 3 码（6 注，12 元）— 与选号页一致

**不是**「百十个各选 3 码共 27 注」，而是 **在 0–9 中选 3 个不同号码**，展开为 **6 注直选全排列**（如 012 → 012/021/102/120/201/210）。不含组三、豹子。

详见 `docs/福彩3D单选复式3码6注回测分析.md`；8538 期全量：

| 策略 | 6注直选中奖 | 命中率 | vs 理论 0.6% |
| --- | ---: | ---: | ---: |
| 融合最优组六 | 59/8538 | **0.691%** | 1.15x |
| 热2+遗漏1 | 56/8538 | 0.656% | 1.09x |
| 遗漏 Top3 | 48/8538 | 0.562% | 0.94x |
| 热号 Top3（近30期） | 45/8538 | 0.527% | 0.88x |
| 随机 3 码 | 45/8538 | 0.527% | 0.88x |

**推荐（2026132 修正版，见下方复盘）**：

| 池 | 3 码 | 6 注 | 预算 |
| --- | --- | --- | ---: |
| **主池** 热2+温1 | **024** | 024 042 204 240 402 420 | 12元 |
| **回补池** 冷温3码 | **347** | 347 374 437 473 734 743 | 12元 |
| 融合组六 | **247** | 247 274 427 472 724 742 | 12元 |

**修正规则（2026131 复盘后）**：

1. **杀号只杀 1 码**：不再杀近30最冷三码；改为上期开奖号中 **近30期整体最热且刚开出** 的 1 码（2026132 杀 **8**）。
2. **三胆改「热2+温1」**：第三位取遗漏 **3–6 期** 的温号，**不用** 遗漏 10+ 的深冷（2026131 的 1 误伤）。
3. **双池并行**：主池热温 + 回补池（近30偏冷、遗漏 1–8 期），各 6 注；24 元比单池 012 覆盖更宽。
4. **组选**：近10期组三 ≤3 期时，主池 **024 组选1注** 可带（2元）。

**注意**：

1. 约 **28%** 期开组三/豹子，此复式 **必不中**，这是玩法结构决定的。
2. 全量优势很弱（最好 1.15x），近 50 期常出现 **0 中**，勿短窗追号。
3. 选号页上的 **已出次数 / 当前遗漏** 对应「近30期频率 + 全位遗漏」；主池用 **热2+温1**，深遗漏放回补池。

## 如何生成预测与复盘

在项目根目录执行：

```text
python src/fc3d_rolling_backtest.py --fixed --data docs/3d.md
python src/fc3d_rolling_backtest.py --data docs/3d.md --report docs/福彩3D逐期自适应回测报告.md
python src/fc3d_strategy_compare.py --data docs/3d.md
python src/fc3d_compound_compare.py --data docs/3d.md
python src/fc3d_straight3_compound.py --data docs/3d.md
```

输出：

- `docs/福彩3D固定权重滚动回测报告.md` — 直选 / 位置命中 / 下一期 Top20
- `docs/福彩3D逐期自适应回测报告.md` — 自适应对照
- `docs/福彩3D多策略回测对比.md` — 热号 / 固定 / 自适应 / 随机，全量及近 500/200/50 期
- `docs/福彩3D组选滚动回测报告.md` — 组选宽口径与可投注口径
- `docs/福彩3D单选复式3码6注回测分析.md` — **选3码6注**（与选号页一致）
- `docs/福彩3D直选复式3码回测分析.md` — 按位3码共27注（不同玩法）

## 赛后复盘清单

每次开奖后回答：

1. 实际号码在融合模型中排名第几？是否在 Top20 / Top100？
2. 没中是因为 **候选池漏号**，还是 **收口删号**（连续 Top 取号过窄）？
3. 三位中有几位被按位热号覆盖？组选宽口径下是否在 20 候选内？
4. 多注是否只是 Top1 附近换壳，有没有真正的跳点票和随机票？
5. 形态（组三/组六）、和值、跨度是否被结构约束误杀？
